hexo 个人博客 学习

发表于 2019-06-09 | 分类于 默认分类 | 浏览数
hexo 个人博客 搭建之后，没有持续 使用导致遗忘干净了，系统重装之后一直也没有再搭建环境，现在使用原来的老环境进行测试

检查hexo环境是否可用

必要前提已经装了 git 和node.js

执行 $ hexo g 报错
   $ bash: hexo: command not found  

解决办法：修改环境变量，在Path里添加node_modules下的.bin文件路径

写博客

  • hexo创建文章
    $ hexo new post "hexo 创建文章"
INFO  Created: E:\IlikeStudy\hexo\source\_posts\hexo-创建文章.md

编写完markdown文件后，根目录下输入hexo g生成静态网页，然后输入hexo s可以本地预览效果，最后输入hexo d上传到github上。

  • 直接在E:\IlikeStudy\hexo\source_posts文件夹下新建.md文件也是可以的，编译并发布到github 命令简写
    hexo g d

注意：每次发布都全部替换

博客目录构成介绍

node_modules是node.js各种库的目录，
public是生成的网页文件目录，
scaffolds里面就三个文件，存储着新文章和新页面的初始设置，
source是我们最常用到的一个目录，里面存放着文章、各类页面、图像等文件，
themes存放着主题文件，一般也用不到。
平时写文章只需要关注source/_posts这个文件夹就行了。
